# NEXT_STEPS.md

Ближайшие технические задачи «АудиоЛад».

## 1. Связать профиль с сессией Supabase (приоритет: высокий)

`/profile` и `/profile/edit` показывают демо-данные. Подключить чтение данных авторизованного пользователя через серверный клиент.

**Файлы:** `src/app/profile/page.tsx`, `src/lib/supabase/server.ts`.

## 2. Закоммитить накопленные изменения (приоритет: средний)

В рабочей копии есть незакоммиченные правки (брендинг, layout, PWA-иконки, `AGENTS.md`, `docs/`, auth-страницы). Согласовать состав коммита с владельцем.

## 3. Добавить защиту приватных маршрутов (приоритет: средний)

Приватные страницы сейчас доступны без авторизации. Реализовать проверку сессии через `proxy.ts` или серверные компоненты.

## 4. Добавить навигацию к auth-страницам (приоритет: низкий)

Ссылки на вход/регистрацию отсутствуют в основном UI. Определить места размещения с владельцем.

## 5. Задокументировать схему базы (приоритет: низкий)

Изучить таблицы, триггеры и RLS через Supabase Studio. Заполнить `docs/DATABASE.md`.

После успешной регистрации выполняется переход на `/auth/sign-in?registered=1`. На странице входа при `registered=1` показывается сообщение об успешной регистрации. Проверено на production.

**Файлы:** `src/app/auth/sign-up/page.tsx`, `src/app/auth/sign-in/page.tsx`.

## Будущий этап: интеграция с MAX (не начат)

Этот этап **не является ближайшей задачей**. Начинается после завершения базовой пользовательской механики:

1. Регистрация и вход.
2. Профиль, связанный с Supabase.
3. Защита приватных маршрутов.
4. Библиотека пользователя с реальными данными.
5. Выдача аудиоматериалов.

### Предварительное исследование (до любой реализации)

- официальный MAX Bot API;
- MAX Mini Apps;
- MAX Bridge;
- серверная валидация initData;
- интеграция BotHelp с собственной системой;
- юридические требования к регистрации и персональным данным.

### Планируемый результат этапа

Воронка: реклама в MAX → бот / BotHelp → бесплатные аудиоматериалы → мини-приложение → серверная проверка данных MAX → поиск или создание профиля → автоматический вход → библиотека пользователя.

Связь идентификаторов: `MAX user_id` ↔ профиль «АудиоЛада» ↔ пользователь Supabase Auth.

Конкретные API, таблицы и схема авторизации будут определены после исследования.
