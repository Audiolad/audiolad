# Listener desktop shell

Техническая фиксация архитектуры будущей desktop-оболочки пользовательской части «АудиоЛад». Документ описывает **план и границы**, а не текущую реализацию UI.

## Концепция

На больших экранах пользовательская часть становится единым приложением:

```text
┌─────────────────────────────────────────────────────────────┐
│                     Верхняя панель                          │
├────────────┬───────────────────────────────┬────────────────┤
│            │                               │                │
│ Моё        │       Основной контент        │ Сейчас играет  │
│ пространство│                              │                │
│            │                               │                │
├────────────┴───────────────────────────────┴────────────────┤
│               Постоянный аудиоплеер                         │
└─────────────────────────────────────────────────────────────┘
```

- **Моё пространство** — левая колонка навигации слушателя.
- **Основной контент** — прокручиваемая центральная область (страницы без смены глобального плеера).
- **Сейчас играет** — правая панель текущей практики (метаданные, следующий шаг, действия).
- **Постоянный нижний плеер** — глобальное управление воспроизведением при переходах между разделами.

## Границы зон

### В listener-shell (будущее)

- `/` — главная
- `/catalog` — каталог
- `/my-practices` — аудиотека
- `/playlists` — плейлисты
- `/history` — история
- `/practice/...` — страницы практик
- `/authors/...` — страницы авторов
- `/p/[slug]` — публичные подборки

### Отдельные пространства (вне listener-shell)

- `/author-dashboard/*` — кабинет автора
- `/admin/*` — админка
- `/auth/*` — вход и регистрация
- `/checkout/*` — оплата
- юридические страницы (`/offer`, `/privacy`, и т.д.)

### Решения отложены

- **`/profile`, `/settings`** — пока не переносятся в shell; отдельное решение позже.
- **`/listen/...`** — пока не переносится и не меняется; полноэкранный мобильный плеер и текущий flow сохраняются.

## Технические принципы

1. **Глобальный `<audio>`** остаётся в root `GlobalAudioPlayerProvider` (`AppProviders`). Shell не владеет audio element.
2. **Страницы** остаются преимущественно Server Components; бизнес-логика и загрузка данных — на уровне page, не shell.
3. **Shell** не дублирует доменную логику страниц; только layout, навигация и client islands (плеер, панель «Сейчас играет»).
4. **Интерактивные части** shell — client islands; оболочка подключается через route group layout без превращения всего приложения в client component.
5. **Мобильный интерфейс** сохраняется: BottomNav и текущий mini-player на малых экранах.
6. **Неработающие функции не показываются** (избранное без БД, офлайн, уведомления, полноценный поиск, похожие практики, таймер сна, UI очереди и т.д.).
7. **Ширина колонок** — фиксированная в v1; ручное изменение ширины — возможное будущее развитие, не первый релиз.

## Foundation (этап 1)

На этапе foundation без видимых изменений UI:

| Артефакт | Назначение |
| -------- | ---------- |
| `src/lib/navigation/listener-nav.ts` | Единый конфиг primary nav (BottomNav сейчас; sidebar позже) |
| `src/app/globals.css` (`--listener-*`) | CSS-переменные shell; `0px` для высот до подключения layout |
| `src/components/listener/ListenerAppShell.tsx` | Пассивный passthrough; **ещё не подключён** в дереве приложения |
