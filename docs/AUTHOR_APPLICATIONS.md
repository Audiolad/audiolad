# AUTHOR_APPLICATIONS.md

Документ описывает текущий поток «Стать автором» в проекте «АудиоЛад».

## Путь автора

1. Пользователь регистрируется как слушатель.
2. Изучает `/become-author` и подаёт заявку.
3. Команда платформы рассматривает заявку вручную.
4. После предварительного одобрения автор готовит профиль и бесплатные материалы.
5. Материалы проходят проверку качества и оформления.
6. Коммерческое размещение подключается отдельно после документов и реквизитов.
7. Автор публикует продукты и развивает аудиторию.

**Важно:** одобрение заявки **не создаёт** автоматически запись в `authors` и `author_members`.

## Статусы заявки

| Статус | Значение |
|--------|----------|
| `draft` | Черновик, можно редактировать |
| `submitted` | Отправлена пользователем |
| `in_review` | На рассмотрении (устанавливает админ) |
| `needs_changes` | Нужны уточнения, пользователь может дополнить |
| `approved` | Одобрена, доступ к кабинету готовится вручную |
| `rejected` | Отклонена |
| `withdrawn` | Отозвана пользователем или админом |

## Таблица `author_applications`

Миграция: `supabase/migrations/20260716170000_author_applications.sql`

Ключевые поля:

- `user_id` — владелец заявки
- `status` — текущий статус
- `display_name`, `direction`, `about`, `planned_content` — содержание заявки
- `links`, `contact`, `has_ready_materials`, `consent_personal_data`
- `submitted_at`, `reviewed_at`, `reviewed_by`, `review_comment` — административные поля

Ограничение: один пользователь может иметь только одну заявку со статусом, отличным от `withdrawn`.

## RLS

- Пользователь видит только свои заявки.
- INSERT/UPDATE только для `user_id = auth.uid()`.
- Триггеры блокируют изменение административных полей и самостоятельную установку статусов `in_review`, `approved`, `rejected`.
- Редактирование содержимого разрешено только в статусах `draft` и `needs_changes`.
- Административное рассмотрение выполняется через `service_role` или SQL от postgres (будущий admin-этап).

## UI

- `/become-author` — посадочная страница и форма заявки
- `/profile` — блок «Стать автором» или статус заявки
- Существующие авторы видят переход в `/author-dashboard`

## Ручной процесс рассмотрения (текущий)

1. Найти заявку в Supabase Studio или через SQL.
2. Изучить содержимое заявки.
3. Обновить статус и комментарий через service/postgres.
4. После финального одобрения создать авторское пространство вручную (шаблон: `supabase/templates/assign_author_owner.sql`).

## Вне scope текущего этапа

- Административная панель заявок
- Автоматическое создание workspace после approval
- Коммерческий договор, реквизиты и выплаты
- Обучение авторов
- Авторская студия
- Email-уведомление владельцу платформы

## Уведомления

Заявки сохраняются в БД. Уведомление на официальный адрес платформы требует отдельного этапа с серверной почтовой инфраструктурой.
