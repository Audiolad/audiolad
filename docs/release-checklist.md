# Release checklist

## Перед деплоем

- [ ] `git status` чистый или осознанный набор изменений
- [ ] `npm run lint`
- [ ] `npm run build` в отдельном релизе, не в `current`
- [ ] commit создан и отправлен в GitHub
- [ ] на сервере достаточно свободного места
- [ ] `shared/.env.production` существует
- [ ] smoke-аккаунт задан через `AUDIOLAD_SMOKE_EMAIL` / `AUDIOLAD_SMOKE_PASSWORD`

## Во время деплоя

- [ ] `deploy.sh` создал новую папку в `releases/`
- [ ] кандидат поднялся на порту `3001`
- [ ] health endpoint кандидата отвечает
- [ ] guest smoke прошёл на кандидате
- [ ] `current` переключён атомарно
- [ ] `previous` указывает на старый релиз
- [ ] PM2 перезагружен один раз
- [ ] production smoke прошёл
- [ ] health-watch 2 минуты прошёл

## После деплоя

- [ ] `release-info.sh`
- [ ] `https://audiolad.ru/` открывается как гость
- [ ] персональная главная не падает после гидратации
- [ ] `/catalog` работает
- [ ] юридическая страница и нижнее меню работают
- [ ] `BUILD_ID` на домене совпадает с `current`

## Если что-то пошло не так

- [ ] не чинить вручную в `current`
- [ ] выполнить `rollback.sh`
- [ ] сохранить логи и скриншоты smoke-теста
- [ ] исправить код в Git и повторить `deploy.sh`
