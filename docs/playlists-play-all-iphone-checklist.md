# Play All — iPhone manual checklist

Проверять на реальном iPhone (Safari) после deploy кандидата.

## Подготовка

1. Owner playlist с: одиночная практика → программа (≥2 аудио) → одиночная практика.
2. Public free playlist с ≥2 playable материалами.
3. Часть unavailable (опционально) для skip toast.

## Сценарии

1. Старт «Слушать всё» с owner detail.
2. Блокировка экрана во время воспроизведения.
3. Pause / Play с lock screen (Media Session).
4. Next / Previous с lock screen.
5. Конец одиночной практики → автостарт следующего продукта.
6. Программа из нескольких треков полностью → следующий продукт.
7. Фон 2–5 минут, затем возврат в Safari.
8. Звук не исчезает после возврата.
9. Metadata (title/artist) обновляется при смене продукта.
10. URL `/listen/...` соответствует текущему продукту (`replace`, без лишней history).
11. Завершение очереди: «Плейлист прослушан».
12. «Прослушать ещё раз» стартует с первого трека первого доступного продукта.
13. «Вернуться к плейлисту» ведёт на `/playlists/[id]` или `/p/[slug]`.
14. Standalone open другого `/listen/...` очищает queue (не продолжает плейлист).
15. Public guest Play All только для free listen-able.
16. Звонок / аудиопрерывание (если безопасно проверить).
17. Bluetooth / AirPods (если доступно).

## Ожидаемые ограничения MVP

- Короткая пауза между продуктами (remount `<audio>`).
- После F5 queue не восстанавливается.
- Gapless / preload нет.
